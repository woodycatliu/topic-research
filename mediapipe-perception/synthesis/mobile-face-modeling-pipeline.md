## 1. 一句話結論
這兩個主題不是平行領域，而是**同一條人臉建模 pipeline 的前後兩半**：MediaPipe 提供「即時、on-device 的感知前端」，3DMM 提供「參數化的建模/重建後端」。

## 2. Pipeline 全景與對應論文

```
相機影像
  │
  ① 人臉偵測（出 ROI）
  │     └─ BlazeFace  1907.05047   200–1000+ FPS，AR 前端
  ▼
  ② 臉部幾何 / landmark（出 2D/3D 點）
  │     └─ Face Mesh  1907.06724   468 頂點 canonical mesh，100–1000+ FPS
  ▼
  ③ landmark ↔ 3DMM 參數（橋）
  │     └─ SynergyNet 2110.09772   landmark 與 3DMM 參數互相回饋
  │     └─ 3DMM-STN   1708.07199   以簡單幾何損失擬合 3DMM 到單張影像
  ▼
  ④ 3D 形狀 / 紋理重建（出可用模型）
  │     └─ Beyond 3DMM 2204.04379  突破 3DMM 表達力、補細節幾何
  │     └─ GCN 高保真  2003.05653  頂點層級補高保真紋理
```

## 3. 兩個對實作最關鍵的概念區分

### (a) Face Mesh ≠ 3DMM —— 互補，非等價
| | MediaPipe Face Mesh (1907.06724) | 3DMM |
| --- | --- | --- |
| 輸出 | 固定拓樸的 **468 點 canonical mesh** | **低維參數係數**（形狀/表情/紋理） |
| 有無統計基底 | 無身分/表情基底 | 有（傳統 PCA，或學習式） |
| 強項 | 即時、貼合、好接 AR | 可控、可編輯、能補遮擋、可參數化 |
| 弱項 | 不可參數化、無語意維度 | 擬合較重 |

→ 多數手機方案是**疊用**：Face Mesh 出穩定 landmark/mesh → 當 3DMM 擬合的約束（即 ③ SynergyNet 那條線）。

### (b) 三層別混為一談
偵測（①ROI）≠ landmark/幾何（②點）≠ 重建（④可用 3D 模型）。SDK 常把前兩層打包，但「建模」通常落在第三、四層。

## 4. 選型決策（先想清楚再寫程式）
- 只要**即時貼合/AR 特效** → 偏前端：BlazeFace + Face Mesh 可能就夠。
- 要**可參數化、可編輯、補遮擋的 3D 臉** → 需後端 3DMM 擬合（③④）。
- 要**兩者兼得** → Face Mesh landmark 當約束餵 3DMM（SynergyNet 思路）。

## 5. 怎麼讀（證據層級不同）
- **MediaPipe 那批（工程技術報告）**：讀「手機上即時、延遲、跨平台的現實約束」，但別當方法學 SOTA（多無嚴謹基準/消融）。
- **3DMM 那批（方法學論文）**：讀「建模/重建理論與品質上限」，注意多數 outline 的量化數字仍「待全文補充」。
