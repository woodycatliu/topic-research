# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 用途

這是一份**研究論文整理規範**。它的目的是引導與說明「papers 該怎麼找、怎麼評估、怎麼歸檔」，定義我要的研究筆記長什麼樣子。

這份文件**不記錄任何當前研究的內容或進度**（不是記憶），只規範「流程與格式」，因此不論研究主題是什麼都適用。

整理產出是繁體中文的 Markdown 文獻筆記，不是程式碼，所以沒有 build / lint / test 流程。

## 工作流程

```
1. 搜尋  → 透過 arXiv MCP（mcp__arxiv-mcp-server__*）依 topic 搜尋論文，
           並把這次的查詢關鍵字/條件記入該主題的 history/。
2. 討論  → 把候選論文提給使用者，一起討論「值不值得收藏」。
           ⚠️ 收藏或捨棄的決定由使用者拍板，不可自行歸檔。
3. 歸檔  → 依討論結果二選一：
           ├─ 值得：建立 <arxiv-id>/ 論文資料夾，寫入連結＋綱要，
           │        並把該篇登錄到此主題的 index.md。
           └─ 不值得：在 rejected/ 新增一篇簡短「為什麼捨棄」，
                    並登錄到 rejected/index.md。
```

## 資料夾結構

固定結構資料夾一律用**英文 kebab-case**；主題資料夾也用英文（例如 `water-thermogenesis/`）。

```
root/
├── index.md                  ← 主題對應表：列出有哪些主題資料夾及其簡述
└── <topic>/                  ← 一個研究主題一個資料夾（英文命名）
    ├── index.md              ← 此主題「已存檔」論文清單
    ├── rejected/             ← 判定不適合此主題的論文
    │   ├── index.md          ← 所有被捨棄論文的標題＋連結（若有）
    │   └── <arxiv-id>.md     ← 每篇一檔，簡短記錄捨棄理由（不需詳細）
    ├── history/              ← 搜尋查詢歷史（關鍵字、查詢條件、時間）
    ├── synthesis/            ← 跨論文的統整 / 綜述 / 整理心得
    └── <arxiv-id>/           ← 每篇收藏論文一個資料夾，以 arXiv ID 命名
        └── outline.md        ← 論文連結 ＋ 綱要
```

## 各檔案格式規範

### `root/index.md`（主題對應表）
- 一列一個主題：主題資料夾名稱、一句話簡述、建立日期。

### `<topic>/index.md`（已存檔論文清單）
- 一列一篇：arXiv ID（連到該論文資料夾）、作者 — *篇名* (年份)、收藏日期。

### `<arxiv-id>/outline.md`（單篇論文）
- **頂部**：論文連結（arXiv、PDF；有 PubMed/PMID 也附上）。
- **綱要**：研究問題、方法、核心發現（數據用表格：項目 / 內容兩欄）、結論。
- **注意事項**：引用時容易誤用或有學術爭議之處（實驗條件、機制辯論、樣本限制）。

### `rejected/<arxiv-id>.md`（捨棄紀錄）
- 簡短即可：標題、連結、**為什麼不適合這個主題**。不需展開內容。

### `history/`（搜尋歷史）
- 記錄每次 arXiv 搜尋的：查詢關鍵字 / 條件、日期、命中數、後續處理（收/棄）。

## 整理原則

- **查證優先**：任何數據（樣本數、百分比、劑量、機制）都須對照可信來源查證並附連結；不可僅憑記憶填入數字。
- **決定權在使用者**：是否收藏、歸到哪個主題，由使用者拍板，不可自行決定後直接歸檔。
- **保留爭議**：結論存在學術辯論時，**兩派觀點都要並列記錄**，不可只取其一。
- **區分證據層級**：原始研究、後續質疑、綜述要分清楚，不可把單一研究的結論當成定論陳述。
- **語言**：固定結構與檔名用英文；筆記內容與回覆一律使用繁體中文。
